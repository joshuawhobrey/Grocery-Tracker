# Grocery-Tracker
Final project from Programming Languages meant to track frequency of groceries purchased, edited after recieving final grade. 

This project was built to help the "Corner Grocer" analyze their daily sales logs so they could optimize their produce section layout. I wrote a C++ program that reads a text file of purchased items, calculates the frequencies, and provides an interactive menu for the manager to search items, view lists, and generate a histogram. I think what I did particularly well was successfully implementing the object-oriented programming requirements. Initially, the hardest part for me was wrapping my head around how to encapsulate the data properly, especially after falling into the "manager class" trap on a previous assignment. I overcame this by closely reviewing my instructor's past feedback and building a dedicated `GroceryItem` class that securely holds the item names and quantities behind private variables. Moving forward, I’m definitely keeping a library of my old project feedback and the official C++ documentation as my main support network.

If I were to enhance the code, I would add strict input validation to the main menu. Right now, if a user accidentally types a letter instead of a number for their menu choice, the input stream could break; fixing that would make the program much more secure and crash-proof. To make the program maintainable and adaptable, I kept all the terminal menus and console print statements entirely within `main.cpp`. Because the `GroceryItem` class doesn't rely on the console at all, the code is clean and could easily be reused if the store ever wanted to upgrade to a graphical user interface later. Ultimately, learning how to completely isolate the user interface from the backend data, along with getting comfortable reading and writing to external files, are the most transferable skills I’ll be taking into my future software projects.
